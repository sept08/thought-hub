# 得到课程评论技能

## 使用场景

当用户阅读完得到课程文章后，帮助用户撰写并提交评论。

## 工作流程

### 1. 收集素材

主动向用户询问以下内容（可根据实际情况调整）：

- **阅读感受**：这篇文章给你带来的整体感受、启发、触动点
- **产生的疑问**：阅读过程中产生的困惑、质疑、想深入探讨的问题
- **交流精华**：我们讨论中达成的共识、精彩的洞见、新的理解
- **行动启发**：这篇文章对你后续行动的影响、想尝试的改变

### 2. 生成评论草稿

基于收集的素材，生成 3-5 种不同风格的评论草稿供用户选择：

**可选风格**：
1. **感悟型**：侧重个人阅读体验和情感共鸣
2. **思辨型**：侧重提出问题和深入思考
3. **行动型**：侧重实践启发和行动计划
4. **综合型**：融合感悟、思考、行动
5. **精华提炼型**：简洁总结核心洞见

每个草稿控制在 200-500 字，符合得到平台的评论风格（真诚、有深度、不空泛）。

### 3. 用户确认（两步确认，不可跳过）

**第一步：选择风格**

使用 AskUserQuestion 工具呈现草稿选项：
- **必须使用 preview 字段展示完整评论内容**，让用户看到将要发布的完整文字
- 用户可选择其中一个风格，或选择 Other 要求修改/融合

**第二步：确认提交（必须执行）**

用户选择风格后，**必须再次询问确认**：
- 展示最终确认的完整评论内容
- 使用 AskUserQuestion 问："以上是最终评论内容，确认提交吗？"
- 用户确认后才能进入提交环节
- 用户可以要求修改，修改后再次确认

**⚠️ 禁止行为**：
- 绝对不能在用户选择风格后直接提交，必须经过第二步确认
- 绝不能跳过"确认提交"这一步

### 4. 提交评论

使用 chrome-devtools-mcp 工具在浏览器中提交：
- 找到评论输入框（通常是 textarea 或特定的评论输入区域）
- 填入确认后的评论内容
- 点击发布按钮

## 评论写作原则

- **真诚**：表达真实的阅读感受，避免套话
- **具体**：引用文章具体内容，避免泛泛而谈
- **有洞见**：展示自己的思考和延伸，不只是复述
- **适度长度**：200-500 字为宜，既充分表达又不冗长
- **互动性**：可以回应作者观点，或与其他读者对话

## 注意事项

- 不要未经确认就提交评论
- 如果用户提供了具体的评论内容，优先使用用户的原话
- 评论发布后告知用户结果
- 如果提交失败，记录原因并告知用户
